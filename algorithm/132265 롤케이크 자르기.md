## 아이디어
Hash(Data Structure)

문제 입력(`topping`)의 길이가 꽤 길다. 무조건 $O(N)$ 이하로 설계를 해야할 듯 하다.

철수와 동생을 사람 A, B라고 하겠다. A, B가 먹는 토핑 가짓수를 측정하려면? 가짓수니까 중복배제 & 속도 → `HashSet<Integer>`

그러면 순차탐색( $O(N)$ )을 하면서 어떻게 A와 B의 토핑을 가를 것인가? A의 Set과 B의 Set에 추가/제거하는 조건이 필요해진다. 토핑들은 중복되서 주어질 수 있고 순차탐색으로 `Set`에 삽입/삭제를 하려면 모든 토핑들의 Maximun index를 구해놓으면 순차탐색으로도 A와 B의 토핑 가지수를 구할 수가 있겠다.
```java
int solution(int[] topping) {
    // 처음에 A는 암것도 못먹고 B는 다 먹는다고 침
    Set<Integer> setA = new HashSet<>();
    Set<Integer> setB = Arrays.stream(topping).boxed().collect(Collectors.toSet());

    // 토핑들의 최대 인덱스를 구해놓음 O(N)
    Map<Integer, Integer> maxIndex = new HashMap<>();
    for (int i = 0; i < topping.length; i++) {
        maxIndex.put(topping[i], i);
    }

    int answer = 0;
    for (int i = 0; i < topping.length; i++) {
        // i번째 index에서 A는 topping[i]가 추가되고
        setA.add(topping[i]);
        // topping[i]가 최대 인덱스라면 B는 더이상 topping[i]를 먹지 못함
        if (maxIndex.get(topping[i]) == i) {
            setB.remove(topping[i]);
        }
        // 가짓수 비교
        if (setA.size() == setB.size()) {
            answer++;
        }
    }
    return answer;
}
```
